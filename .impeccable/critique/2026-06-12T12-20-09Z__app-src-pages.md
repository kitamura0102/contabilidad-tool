## Anti-Patterns Verdict

**LLM assessment:** This does NOT read as obvious AI slop. The density, monospace data columns, and slate/blue system feel like a real professional tool — earned familiarity, which is the right bar for product UI. The one tell is the **four identical stat cards** at the top of the dashboard: icon + label + big number + sub-line, repeated 4×. That's the SaaS-dashboard reflex. It's not banned by your chosen anti-references, but 2 of the 4 metrics are vanity ("Clientes activos" duplicates the count already in the subtitle; "Facturas totales" isn't actionable).

**Deterministic scan:** 1 finding — `overused-font` (Geist) at `index.html:10`, severity warning. For product UI this is acceptable (the product register explicitly permits familiar sans), so it's low priority. No side-stripes, gradient text, or glassmorphism detected.

**Visual overlays:** Not available — the surface is auth-gated and no browser overlay could be injected. Fallback: source review.

## Overall Impression

This is a genuinely good product surface — the strongest thing about it is that it disappears into the task. An accountant would trust it. The biggest opportunity isn't aesthetic, it's **fit to the actual workflow**: accountants work in batch at month-end, and right now everything is one-at-a-time. The second gap is accessibility — a data-dense table tool where the table headers fail contrast and the rows can't be keyboard-navigated.

## What's Working

- **Domain fluency.** RNC/NCF/ITBIS/606/607, monospace for all identifiers and amounts, Spanish error mapping. This is the "precisión" principle made real.
- **Status legibility.** Badges combine color + icon + text (never color alone), and the auto-refresh-every-10s indicator with live spinner is exactly the "calm under load" principle.
- **Density done right.** Table rows with totals footer, tabs with counts, dense toolbar — respects the professional user instead of padding everything into cards.

## Priority Issues

- **[P1] Table headers and faint text fail WCAG AA contrast.** `slate-400` (#97A1AD) measures **2.46:1** on the `slate-50` header background — well under the 4.5:1 floor for small text. This color is also used for sector subtitles, the `muted-cell` dashes, and input placeholders. In a tool whose entire job is reading data accurately, the column labels being hard to read is a real defect.
  - **Fix:** Bump table-header / placeholder text from `slate-400` to `slate-500` (#67727E, 4.61:1) minimum, ideally `slate-600`. Keep `slate-400` only for genuinely decorative glyphs (em-dashes).
  - **Suggested command:** `/impeccable audit`

- **[P1] Table rows aren't keyboard accessible.** Both tables use `<tr onClick={...}>` with no `tabIndex`, `role`, or key handler. The primary navigation action — opening a client or a factura — is mouse-only. Power users (Alex) and keyboard/screen-reader users (Sam) are blocked from the core flow.
  - **Fix:** Make rows focusable (`tabIndex={0}`, `role="button"`), handle Enter/Space, add a visible `:focus-visible` ring. Or render the row's primary cell as a real link/button.
  - **Suggested command:** `/impeccable audit`

- **[P1] No `prefers-reduced-motion` support.** `index.css` ships `.spin`, `cf-slide-in`, `cf-rise`, `cf-fade`, `cf-pulse` and several transitions, with zero reduced-motion fallback. Users with vestibular sensitivity get the full spinner + slide-in treatment.
  - **Fix:** Add a `@media (prefers-reduced-motion: reduce)` block that drops transforms to opacity-only / instant.
  - **Suggested command:** `/impeccable audit`

- **[P2] Workflow is one-at-a-time; the user works in batch.** Upload takes a single file; there's no multi-select on facturas, no batch "Reintentar," no select-all. Month-end means dozens of invoices — your own PRODUCT.md says "processing invoices in batch." The UI doesn't match.
  - **Fix:** Multi-file upload, row checkboxes with a bulk action bar (reintentar / delete / mark reviewed).
  - **Suggested command:** `/impeccable shape` (this is a feature, not a paint job)

- **[P2] Loading is plain text, not skeletons.** Dashboard shows "Cargando…" centered. The product register calls for skeleton states over text/spinners in content. A skeleton of the table preserves layout and feels faster.
  - **Suggested command:** `/impeccable harden`

## Persona Red Flags

**Alex (Power User):** No keyboard shortcuts anywhere. Can't Tab to a table row and hit Enter. Can't bulk-reintentar the 6 errored facturas — must click each row, then the button, 6 times. No multi-file upload. Every repetitive action is full-mouse, full-manual.

**Sam (Accessibility):** Table headers at 2.46:1 are below the readable floor. Rows are `<tr onClick>` — invisible to keyboard and announced as plain table cells by a screen reader, not as actionable. No focus indicators on rows. The corrector's image panel has no alt strategy for the PDF iframe.

**Carmen (the Dominican contador, month-end — project persona):** Has 8 clients, ~40 invoices each, due to DGII in two days. Opens a client, sees 6 "Revisar." Has to open each one individually in the full-screen corrector, fix, save, return, repeat. No queue/next-button to move through the review list. No way to see across all clients "what's still blocking my exports" — she has to visit each client to find out. The dashboard's "Por revisar: N" is a number, not a worklist she can click into.

## Minor Observations

- Corrector header has both a back-arrow (`ArrowLeft`) and an `X`, both calling `closePanel()`. Redundant — pick one.
- Left panel of the corrector uses hardcoded `#1A1F27` instead of a token (`--slate-900` is #161B22, close but not it).
- `z-index` scale is semantic-ish but `.flow` (40) sits below `.overlay` (50); they don't co-occur today, so it's latent, not a live bug.
- Period selector overlays icons on a native `<input type="month">`; alignment may drift across browsers.
